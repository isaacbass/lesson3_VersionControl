Lesson 3 Highlights

Isaac Bass

Highlight #1 Version Control

Version control is a system that records changes to a file or set of files over time so that you can recall specific versions later. A version control system (VCS) is ideal for graphics and web designers, who typically need to keep every version of an image or layout. A VCS allows the user to revert files back to a previous state, revert an entire project back to a previous state, compare changes made over time, see who last made a modification, and more.

Highlight #2 Snapshots

The main difference between Git and other VCSs is the way Git thinks about its data. Conceptually, most other VCSs store information as a list of file-based changes. Git thinks of its data more like a set of snapshots of a miniature filesystem. Every time you commit, Git basically takes a picture of what all your files look like at that moment and stores a reference to that snapshot. Git thinks about its data more like a stream of snapshots.

Highlight #3 Git Repositories

There are two main approaches you can use to get a Git project. The first takes an existing project or directory and imports it into Git. If you're starting to track an existing project in Git, you need to go to the project's directory and type "git init." This will create a new subdirectory named .git that contains all of your necessary repository files â€” a Git repository skeleton. The second clones an existing Git repository from another server. To get a copy of an existing Git repository the command you need is "git clone."

Highlight #4 Tracked vs. Untracked Files

Each file in your working directory can be in one of two states: tracked or untracked. Tracked files are files that were in the last snapshot, and they can be modified, unmodified, or staged. Untracked files are everything else. Untracked files are any files in your working directory that were not in your last snapshot and are not in your staging area. When you first clone a repository, all of your files will be tracked and unmodified because Git just checked them out and you haven't edited anything.

Highlight #5 Three States

Git has three main states that files can reside in: committed, modified, and staged. Committed files are safely stored in the user's local database. Modified files have been changed but not committed to the user's database yet. Staged files are modified files that have been marked in their current version to go into the user's next commit snapshot.
